# Documents API

Base path:

```text
/api/v1/knowledge-bases/{knowledge_base_id}/docs
```

Documents are **platform-authored resources** nested under a knowledge base. Like knowledge
bases, they have no tenant relationship. All endpoints require a platform role and the
relevant `doc:*` permission, plus resource-level authorization on the parent knowledge base
(see [knowledge-bases.md](./knowledge-bases.md)).

Uploads use **MinIO/S3 direct upload**: the API mints a short-lived presigned URL, the client
PUTs the file straight to object storage, and the API only manages control-plane metadata. The
API never receives the file body. There are no top-level `/documents/*` or `/uploads/*` routes.

Object key format (generated by the backend, never the client):

```text
knowledge-bases/{knowledge_base_id}/documents/{document_id}/{safe_file_name}
```

Allowed file types (whitelist): `pdf`, `docx`, `pptx`, `xlsx`, `md`, `txt`, `html`, `csv`, `json`.

## POST /upload-url

Request a presigned upload URL. Required permission: `doc:create`.

Request:

```json
{
  "fileName": "guide.pdf",
  "mimeType": "application/pdf",
  "fileSizeBytes": 102400,
  "contentHash": "sha256_optional"
}
```

The file type is resolved from the extension against the whitelist; unsupported types are
rejected (`400 DOC_UNSUPPORTED_TYPE`).

Response:

```json
{
  "uploadSessionId": "upl_123",
  "documentId": "doc_123",
  "method": "PUT",
  "uploadUrl": "https://minio.example.com/expert-docs/...",
  "headers": { "Content-Type": "application/pdf" },
  "objectKey": "knowledge-bases/kb_123/documents/doc_123/guide.pdf",
  "expiresAt": "2026-06-03T10:10:00Z"
}
```

The client then `PUT`s the file body to `uploadUrl` directly (the API does not proxy it).

## POST /upload-urls

Request presigned upload URLs for multiple documents. Required permission: `doc:create`.

Request:

```json
{
  "files": [
    {
      "fileName": "guide.pdf",
      "mimeType": "application/pdf",
      "fileSizeBytes": 102400
    },
    {
      "fileName": "notes.txt",
      "mimeType": "text/plain",
      "fileSizeBytes": 4096
    }
  ]
}
```

Response:

```json
{
  "items": [
    {
      "uploadSessionId": "upl_123",
      "documentId": "doc_123",
      "method": "PUT",
      "uploadUrl": "https://minio.example.com/expert-docs/...",
      "headers": { "Content-Type": "application/pdf" },
      "objectKey": "knowledge-bases/kb_123/documents/doc_123/guide.pdf",
      "expiresAt": "2026-06-03T10:10:00Z"
    }
  ]
}
```

The client then `PUT`s each file body to its corresponding `uploadUrl`.

## POST /complete-upload

Confirm the upload and create the document record. Required permission: `doc:create`.

Request:

```json
{
  "uploadSessionId": "upl_123",
  "etag": "minio-etag",
  "fileSizeBytes": 102400
}
```

The API HEADs the object and **enforces the declared size**: the stored object size must equal
`upload_sessions.file_size_bytes` recorded at upload-url time. A presigned PUT does not bound
the written size, so the HEAD size is authoritative — the client-provided `fileSizeBytes` is
never trusted. On mismatch the session is failed, the object is removed, no document is created,
and `400 UPLOAD_SIZE_MISMATCH` is returned.

`etag` is the object MD5 for a single PUT (not the sha256 `contentHash`), so content integrity
is verified later at build time, not here.

Response `201 Created`:

```json
{
  "id": "doc_123",
  "knowledgeBaseId": "kb_123",
  "fileName": "guide.pdf",
  "fileType": "pdf",
  "mimeType": "application/pdf",
  "fileSizeBytes": 102400,
  "storageKey": "knowledge-bases/kb_123/documents/doc_123/guide.pdf",
  "contentHash": null,
  "parseStatus": "pending",
  "indexStatus": "pending",
  "metadata": {},
  "createdAt": "2026-06-03T10:00:00Z",
  "updatedAt": "2026-06-03T10:00:00Z"
}
```

## GET /

List documents in a knowledge base. Required permission: `doc:read`. Soft-deleted documents are
excluded.

```json
{ "items": [ { "id": "doc_123", "...": "..." } ] }
```

## GET /{document_id}

Get a document. Required permission: `doc:read`.

## PATCH /{document_id}

Update document metadata / file name. Required permission: `doc:update`.

```json
{ "fileName": "renamed.pdf", "metadata": {} }
```

## DELETE /{document_id}

Soft-delete a document. Required permission: `doc:delete`. Response: `204 No Content`.

The delete is effective immediately: the row is marked `deleted_at` and excluded from all
queries, and retrieval (chat/search) must filter `deleted_at is null`. The underlying MinIO
object is removed asynchronously by a background GC pass (by `storage_key`; the client-supplied
key is never trusted).

## GET /{document_id}/download-url

Mint a short-lived presigned GET URL for the document object. Required permission: `doc:read`.

```json
{
  "method": "GET",
  "downloadUrl": "https://minio.example.com/expert-docs/...",
  "expiresAt": "2026-06-03T10:10:00Z"
}
```
