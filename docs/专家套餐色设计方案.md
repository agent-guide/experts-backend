专家系统套餐管理方案
专家、Skill、知识库的归属关系
对象	谁创建	谁维护	用户是否能直接创建	套餐应该怎么控制
专家	专家系统管理员	专家系统管理员	一般不能	控制用户可访问哪些专家、可使用几个专家
Skill	专家系统管理员	专家系统管理员	一般不能	不建议套餐直接暴露 Skill，通常跟随专家授权
知识库	专家系统管理员	专家系统管理员上传文件维护	一般不能	不建议用户直接选择知识库，通常跟随专家授权
模型	系统平台	系统平台	不能	控制可调用模型等级
问答次数 / Token	系统统计	系统统计	不涉及	按套餐限制额度
席位	用户 / 企业管理员邀请	用户 / 企业管理员	可以邀请成员	按套餐限制团队人数
注：问答次数/Token有一个触达上限，则表明套餐用量已用完。

套餐绑定“专家分组”，专家加入分组，套餐通过分组获得专家访问权限。
套餐	可访问专家分组

专家权限链路
用户当前套餐是否有效
↓
套餐是否启用或用户是否仍在有效订阅期
↓
套餐是否授权该专家或该专家所属分组
↓
该专家绑定的模型是否在套餐允许范围内
↓
用户问答次数 / Token 是否足够
↓
允许召唤专家

套餐升级规则
系统只允许从低等级套餐升级到高等级套餐，不允许直接在升级入口做降级。

当前套餐	可升级套餐

专业版	Max版、Business版
Max版	Business版
Business版	无更高套餐

2. 升级逻辑
免费版升级
免费版用户升级为任意付费套餐时：
选择目标套餐
选择月付或年付
进入支付页面
支付成功后立即生效
原免费权益替换为新套餐权益

付费套餐升级
例如：专业版升级为 Max版。
建议支持的规则：
规则	说明
立即升级，按差价补款	用户补当前周期剩余时间的差价

剩余价值约 ￥499.5。

推荐交互规则
“最受欢迎”标签规则
建议系统限制：
同一时间只能有一个套餐被标记为“最受欢迎”。
如果管理员设置专业版为最受欢迎，其他套餐自动取消该标签。

删除套餐规则
免费版作为默认套餐，不建议删除，只允许编辑部分权益。

已有订阅用户的套餐不可直接删除
当套餐下存在订阅用户时，不允许删除该套餐，只允许停用。
套餐停用后，该套餐不再对新用户展示，不允许新购，不允许作为升级目标，也不允许续费。
已订阅该套餐且仍在有效期内的用户，可继续使用当前周期内的套餐权益。
套餐到期后，系统自动将用户切换至免费版，或引导用户选择其他启用套餐。
所有历史订单、订阅记录和用量数据必须保留，不得删除。

套餐停用后的逻辑
场景	处理方式
新用户购买	不展示，不可购买
老用户当前周期	不影响，继续使用到期
老用户续费	不允许续费，引导迁移到其他套餐
用户升级	不允许升级到已停用套餐
用户从停用套餐升级	允许升级到更高或可用套餐
后台查看	保留套餐和历史订单
数据统计	保留使用数据和订阅记录

套餐到期后的建议
1. 到期前 7 天提醒用户当前套餐即将停用。 
2. 提供可迁移套餐。 
3. 到期后如果没有选择新套餐，再降级到免费版。 
4. 降级后保留历史使用数据，不删除用户记录。 
5. 如果用户原套餐绑定了高级专家，到期后这些专家变为不可用状态。 

推荐文案示例：
当前套餐已停止销售，将无法继续续费。你可以在到期前选择新的套餐，当前权益可使用至 2027-06-05。
